# User Flow Patterns

**Purpose:** Design intuitive user journeys that guide users efficiently through common tasks.

---

## What is a User Flow?

A user flow is the path a user takes to complete a task. Good flows:
- Minimize steps
- Provide clear progress
- Prevent errors
- Allow recovery

### Onboarding Guidelines

- **Keep it short:** 3-5 screens max
- **Show value:** Explain benefits, not features
- **Allow skip:** Some users want to explore first
- **Progressive:** Don't ask for everything upfront
- **Contextual:** Request permissions when relevant

### CRUD Guidelines

- Confirm destructive actions
- Provide undo when possible
- Show success feedback
- Validate before submission
- Save drafts automatically
- Handle conflicts (concurrent editing)

---

## Flow Design Best Practices

### 1. Define Entry Points
Where do users start this flow?

### 2. Map the Happy Path
What's the ideal journey?

### 3. Handle Edge Cases
- What if they go back?
- What if there's an error?
- What if they're interrupted?

### 4. Provide Escape Routes
- Can users cancel?
- Can they save and continue later?
- Can they skip optional steps?

### 5. Show Progress
- Where am I in the flow?
- How much is left?
- Can I see what I've completed?

### 6. Minimize Steps
- Can any steps be combined?
- Can any steps be eliminated?
- Are all fields necessary?
