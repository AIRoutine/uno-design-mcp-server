## Accessibility Requirements

### WCAG Guidelines

| Requirement | Value | Standard |
|-------------|-------|----------|
| Minimum target size | 44x44 CSS px | WCAG 2.1 AAA |
| Target spacing | 24 CSS px | WCAG 2.2 |
| Pointer target size | 24x24 CSS px | WCAG 2.2 AA |

### Platform Accessibility

| Platform | Accessibility Setting | Effect |
|----------|----------------------|--------|
| iOS | Larger Text | Scales UI elements |
| Android | Display Size | Scales touch targets |
| Windows | Make everything bigger | Scales all elements |

## XAML Implementation

```xml
<!-- Touch-friendly button -->
<Button
    Content="Submit"
    MinWidth="64"
    MinHeight="48"
    Padding="16,0"/>

<!-- Icon button with adequate hit area -->
<Button
    MinWidth="48"
    MinHeight="48"
    Padding="12"
    Background="Transparent">
    <FontIcon Glyph="&#xE10F;" FontSize="24"/>
</Button>

<!-- Touch-friendly checkbox -->
<CheckBox
    Content="Remember me"
    MinHeight="48"
    Padding="12,0,0,0"/>

<!-- List item with full-width touch -->
<ListViewItem MinHeight="56" Padding="16,8">
    <StackPanel Orientation="Horizontal">
        <Image Width="40" Height="40"/>
        <TextBlock Text="Item Title" Margin="16,0,0,0"/>
    </StackPanel>
</ListViewItem>
```

## Checklist

- [ ] All touch targets are at least 44x44dp (48x48dp preferred)
- [ ] All click targets are at least 24x24px (32x32px preferred)
- [ ] Minimum 8dp spacing between adjacent touch targets
- [ ] Primary actions are in thumb-friendly zones on mobile
- [ ] Visual size can be smaller than hit area
- [ ] List items are full-width tappable
- [ ] Form inputs meet minimum height requirements
- [ ] Icons have adequate padding for touch
- [ ] Tested with accessibility settings enabled
