# Design Review Checklists

**Purpose:** Systematic checklists to ensure design quality at every phase.

---

## Wireframe Review Checklist

### Structure
- [ ] Information hierarchy is clear
- [ ] Navigation structure is defined
- [ ] All main user flows are mapped
- [ ] Content areas are appropriately sized
- [ ] Key actions are prominently placed

### Layout
- [ ] Grid system is consistent
- [ ] Spacing follows a system (4/8px multiples)
- [ ] Content is properly aligned
- [ ] White space is intentional
- [ ] Layout works for all content lengths

### User Flow
- [ ] Entry points are clear
- [ ] Navigation between screens is logical
- [ ] Back/exit options are available
- [ ] Error states are considered
- [ ] Empty states are designed

### Responsiveness
- [ ] Mobile layout is considered
- [ ] Tablet layout is considered
- [ ] Desktop layout is considered
- [ ] Breakpoint behavior is defined
- [ ] Content priority is maintained across sizes

---

## Visual Design Review Checklist

### Color
- [ ] Color palette is defined and consistent
- [ ] Text contrast meets WCAG AA (4.5:1)
- [ ] UI component contrast meets WCAG (3:1)
- [ ] Colors have consistent meaning
- [ ] Dark mode colors are considered
- [ ] Color is not the only indicator

### Typography
- [ ] Type scale is defined and followed
- [ ] Body text is minimum 16px
- [ ] Line height is appropriate (1.4-1.6 body)
- [ ] Line length is readable (50-75 chars)
- [ ] Font weights create clear hierarchy
- [ ] No more than 2-3 fonts used

### Spacing & Layout
- [ ] Spacing is consistent throughout
- [ ] Alignment follows grid system
- [ ] Visual grouping is clear
- [ ] White space supports hierarchy
- [ ] Elements don't feel cramped

### Visual Elements
- [ ] Icons are consistent style
- [ ] Images have appropriate treatment
- [ ] Elevation/shadows are consistent
- [ ] Border radius is standardized
- [ ] Visual style matches brand

### Components
- [ ] Button hierarchy is clear
- [ ] Input states are all designed
- [ ] Cards are consistent
- [ ] Lists are well-structured
- [ ] All component states exist

---

## Prototype Review Checklist

### Interactions
- [ ] Click/tap targets are defined
- [ ] Hover states are designed (desktop)
- [ ] Press states are designed
- [ ] Focus states are visible
- [ ] Disabled states are clear

### Animations & Transitions
- [ ] Transitions are smooth (200-300ms)
- [ ] Animations support understanding
- [ ] Motion is not excessive
- [ ] Loading states are present
- [ ] Micro-interactions are consistent

### Navigation
- [ ] All links/buttons work
- [ ] Back navigation is functional
- [ ] Deep linking is considered
- [ ] Navigation is consistent
- [ ] Current location is indicated

### Feedback
- [ ] Actions have immediate feedback
- [ ] Loading states are shown
- [ ] Success states are confirmed
- [ ] Error states are helpful
- [ ] Undo options exist where needed

### Edge Cases
- [ ] Empty states are handled
- [ ] Error states are handled
- [ ] Long content is handled
- [ ] Short content is handled
- [ ] Offline states are considered

---

## Final Review Checklist

### Accessibility
- [ ] Color contrast passes WCAG AA
- [ ] Touch targets are minimum 44x44
- [ ] Focus indicators are visible
- [ ] Screen reader labels exist
- [ ] Keyboard navigation works
- [ ] Motion can be reduced
- [ ] Text can be resized

### Usability
- [ ] Primary actions are obvious
- [ ] Navigation is intuitive
- [ ] Forms are easy to complete
- [ ] Errors are preventable
- [ ] Help is available
- [ ] Feedback is immediate

### Consistency
- [ ] Visual style is consistent
- [ ] Interaction patterns are consistent
- [ ] Terminology is consistent
- [ ] Spacing is consistent
- [ ] Component usage is consistent

### Performance
- [ ] Images are optimized
- [ ] Animations are smooth
- [ ] Load times are reasonable
- [ ] Skeleton screens exist
- [ ] Progress indicators exist

### Platform Guidelines
- [ ] iOS HIG compliance (if iOS)
- [ ] Material Design compliance (if Android)
- [ ] Fluent Design compliance (if Windows)
- [ ] Platform patterns are followed
- [ ] Platform navigation is respected

### Content
- [ ] Copy is clear and concise
- [ ] Labels are descriptive
- [ ] Error messages are helpful
- [ ] Empty states have guidance
- [ ] Microcopy is consistent

## Quick Pre-Handoff Checklist

Before handing off to development:

### Specifications
- [ ] All colors are documented
- [ ] Typography scale is documented
- [ ] Spacing values are documented
- [ ] Component specs are complete
- [ ] Breakpoints are defined

### Assets
- [ ] Icons are exported
- [ ] Images are optimized
- [ ] Assets are organized
- [ ] Naming is consistent

### States
- [ ] All interactive states documented
- [ ] Loading states provided
- [ ] Error states provided
- [ ] Empty states provided
- [ ] Edge cases covered

### Annotations
- [ ] Behaviors are explained
- [ ] Animations are specified
- [ ] Conditional logic is documented
- [ ] Responsive behavior is clear

## Review Meeting Template

### Before Review
1. Define review scope
2. Share designs in advance
3. Identify key questions

### During Review
1. Present context and goals
2. Walk through flows
3. Highlight decisions and tradeoffs
4. Gather feedback systematically

### After Review
1. Document feedback
2. Prioritize changes
3. Assign action items
4. Schedule follow-up if needed
