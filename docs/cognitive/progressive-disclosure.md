## Anti-Patterns

### 1. Hiding Essential Features
```
BAD: Main navigation behind hamburger on desktop
GOOD: Only collapse on mobile where space is limited
```

### 2. Too Many Clicks to Reach Common Features
```
BAD: Settings > Advanced > More > Other > The thing you need
GOOD: Frequently used features at Layer 1 or 2 maximum
```

### 3. No Indication of Hidden Content
```
BAD: Features exist but users don't know to look for them
GOOD: Clear "More options" or "â–¸" indicators
```

### 4. Inconsistent Disclosure Patterns
```
BAD: Some sections expand, others go to new page, others show modal
GOOD: Consistent pattern throughout the app
```

---

## Checklist

- [ ] Core features are visible without interaction
- [ ] Advanced features are discoverable (clear indicators)
- [ ] Disclosure controls are consistent
- [ ] Hidden content is accessible within 1-2 clicks
- [ ] Mobile appropriately collapses desktop features
- [ ] Power users can access advanced features quickly
- [ ] New users aren't overwhelmed on first use
